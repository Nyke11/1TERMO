📋 Engenharia de Requisitos: Guia de Aula
Este guia é o seu roteiro para dominar a arte de entender o que o cliente precisa e transformar desejos em sistemas reais e funcionais. 🏗️✨

1. 📂 O Coração do Projeto: Tipos de Requisitos
Saber separar o "o quê" do "como" é o primeiro passo para um software de sucesso.

✅ Requisitos Funcionais (RF)
Descrevem as funções que o sistema deve executar. É o comportamento esperado.

Exemplo: "O sistema deve permitir o cadastro de novos usuários." 👤

Exemplo: "O sistema deve gerar um relatório mensal de vendas em PDF." 📄

🛠️ Requisitos Não Funcionais (RNF)
Descrevem as qualidades e restrições do sistema. Não é o que ele faz, mas como ele é.

Exemplo (Desempenho): "As páginas devem carregar em menos de 2 segundos." ⚡

Exemplo (Segurança): "As senhas devem ser armazenadas com criptografia SHA-256." 🔐

Exemplo (Usabilidade): "O sistema deve ter interface responsiva para celulares." 📱

2. 🧠 Técnicas de Elicitação (Descobrindo a Verdade)
Como extrair as informações corretas dos stakeholders (interessados).

⚡ Brainstorm: Reunião criativa para gerar ideias sem filtros. Ótimo para descobrir funções inovadoras. 💡

🎤 Entrevistas: Conversa direta para entender as dores do usuário. Foque em perguntas abertas: "Como você resolve isso hoje?" 🗣️

3. 🎨 Prototipagem e Diagramas (Visualizando a Ideia)
Antes de escrever código caro, desenhamos modelos baratos para validar o caminho.

📐 Diagramas (UML)
Casos de Uso: Um mapa que mostra quem são os atores e quais ações eles fazem no sistema. 👤➡️⚙️

🖌️ Protótipos
Baixa Fidelidade: Esboços rápidos em papel ou wireframes. Foco na lógica, não na beleza. 📝

Alta Fidelidade: Telas que parecem o sistema real (ex: Figma). Ótimo para validar a experiência do usuário (UX). ✨

4. 📄 Relatórios Técnicos e Documentação
Se não está escrito, não existe! A formalização protege a equipe e o cliente.

📝 Especificação de Requisitos (ERS): O documento mestre que lista todos os RFs e RNFs detalhadamente.

📊 Relatório de Viabilidade: Analisa se temos tempo, dinheiro e tecnologia para realizar o projeto. ⚖️

🔗 O Fluxo do Levantamento
Um processo de requisitos bem estruturado segue este caminho:

Elicitação 🧠 -> Coleta de ideias (Entrevistas/Brainstorm).

Análise 🔍 -> Separar o que é funcional do que é não funcional.

Especificação 📄 -> Escrita do documento técnico (ERS).

Validação ✅ -> Apresentação do Protótipo ao cliente para aprovação.
